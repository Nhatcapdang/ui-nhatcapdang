## Overview

VelocityText is a client-side React component that creates dynamic text animations based on scroll velocity. As users scroll, the text moves horizontally with a skew effect that responds to scrolling speed, creating an engaging visual experience.

## Features

- **Scroll-based Animation**: Text movement triggered by scroll position
- **Velocity-responsive Skew**: Skew effect based on scroll velocity
- **Customizable Repeat**: Display multiple instances of the content
- **Reversible Animation**: Support for forward and reverse animations
- **Spring Physics**: Smooth animations using spring dynamics
- **Fully Typed**: Written in TypeScript with full type safety

## Examples

### Basic Usage

The simplest implementation with default settings:

```tsx
<VelocityText>
	Velocity Text Animation
</VelocityText>
```

### Repeated Content

Create a marquee-like effect by repeating the content multiple times:

```tsx
<VelocityText repeat={8}>
	Scroll to see the effect •{' '}
</VelocityText>
```

### Reverse Animation

Reverse the animation direction:

```tsx
<VelocityText repeat={6} reverse>
	This text moves in reverse •{' '}
</VelocityText>
```

### Custom Styling

Add custom styles using the className prop:

```tsx
<VelocityText
	repeat={5}
	className="text-4xl font-bold text-blue-600 my-20"
>
	Custom Styled Text •{' '}
</VelocityText>
```

### Multiple Sections

Create alternating forward and reverse sections:

```tsx
<div className="space-y-8">

	</VelocityText>

	<VelocityText repeat={8} reverse className="text-2xl">
		Reverse Animation •{' '}
	</VelocityText>

	<VelocityText repeat={8} className="text-2xl">
		Forward Again •{' '}
	</VelocityText>
</div>
```

## How It Works

The VelocityText component uses several Framer Motion hooks to create smooth, physics-based animations:

1. **useScroll**: Tracks scroll progress within a specific viewport range (`start 90%` to `start 10%`)
2. **useVelocity**: Calculates the velocity of scrolling
3. **useTransform**: Maps scroll values to animation properties (position and skew)
4. **useSpring**: Applies spring physics for smooth, natural-feeling animations

### Animation Parameters

The component uses these spring physics parameters:

- **mass**: `3` - Affects the weight/inertia of the animation
- **stiffness**: `400` - Controls how quickly the spring responds
- **damping**: `50` - Controls the bounce/oscillation of the spring

## Technical Details

### Scroll Offset

The component starts animating when the element reaches `90%` of the viewport and stops when it reaches `10%`:

```tsx
offset: ['start 90%', 'start 10%']
```

### Skew Range

The skew effect ranges from `-45deg` to `45deg` based on scroll velocity:

```tsx
[-0.5, 0.5] → ['45deg', '-45deg']
```

### Position Range

The horizontal position animates from `-4000px` to `0px`:

```tsx
[0, 1] → [-4000, 0]
```

## Best Practices

1. **Add Spacing**: Include separator characters (like `•`) between repeated content for visual clarity
2. **Responsive Design**: Adjust repeat count based on viewport size for optimal display
3. **Performance**: Use this component sparingly as scroll-based animations can impact performance
4. **Content Length**: Keep content concise for better visual effect
5. **Color Contrast**: Ensure text has sufficient contrast with the background

## Browser Support

This component requires modern browser support for:

- CSS transforms
- JavaScript scroll events
- Framer Motion compatibility

Works in all modern browsers (Chrome, Firefox, Safari, Edge).

## Accessibility

The component is marked as `'use client'` and relies on visual effects. Consider these accessibility aspects:

- The text remains readable even with skew effects
- Content is part of the DOM and accessible to screen readers
- Consider providing alternative content for users with motion sensitivity
- Test with `prefers-reduced-motion` media query if needed

## Notes

- This is a client component (`'use client'`) and cannot be used in Server Components
- The component requires a `cn` utility function for className merging
- Scroll detection works within a specific viewport range, so positioning matters
- Initial position is off-screen (`-4000px`), animating to `0px` as you scroll
