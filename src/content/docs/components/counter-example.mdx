## Available Counter Variants for MDX

The following counter variants are available for use in MDX files:

### BasicCounter

Standard number formatting with comma separators:

```mdx
<BasicCounter targetValue={1250} />
<!-- Result: 1,250 -->
```

### CurrencyCounter

USD currency formatting:

```mdx
<CurrencyCounter targetValue={45000} />
<!-- Result: $45,000 -->
```

### PercentageCounter

Percentage with no decimals:

```mdx
<PercentageCounter targetValue={98} />
<!-- Result: 98% -->
```

### DecimalPercentageCounter

Percentage with one decimal place:

```mdx
<DecimalPercentageCounter targetValue={98.7} />
<!-- Result: 98.7% -->
```

## Custom Formatters (React/TypeScript Only)

For React/TypeScript files, you can create custom formatters:

### K/M Notation

```tsx
import Counter from '@/components/counter';

const kFormatter = (value: number) => {
  if (value >= 1000000) return `${(value / 1000000).toFixed(1)}M`;
  if (value >= 1000) return `${(value / 1000).toFixed(1)}K`;
  return value.toString();
};

<Counter targetValue={1500000} format={kFormatter} />;
// Displays as "1.5M"
```

### Time Format

```tsx
const timeFormat = (value: number) => {
  const hours = Math.floor(value);
  const minutes = Math.round((value - hours) * 60);
  return `${hours}h ${minutes}m`;
};

<Counter targetValue={42.5} format={timeFormat} />;
// Displays as "42h 30m"
```

**Note**: Custom formatters with inline functions cannot be used in MDX files due to React Server Component limitations. Create a new counter variant in `/src/components/counter-variants.tsx` if you need additional formatters for MDX.

## Tips for Best Results

### Animation Speed

The counter uses spring physics. For faster animations, you can modify the component's spring settings:

- Lower stiffness = slower, more bouncy
- Higher stiffness = faster, more rigid
- Higher damping = less bounce

### Viewport Trigger

The counter triggers when it enters the viewport. Keep in mind:

- Animation happens only once (`once: true`)
- Triggers immediately when in viewport
- Use `delay` prop to add pause before starting

### Visual Design

Best practices for counter display:

1. **Size Appropriately**: Larger numbers for primary metrics
2. **Add Context**: Include labels and descriptions
3. **Use Color**: Highlight with theme colors
4. **Group Related**: Display similar metrics together
5. **Add Icons**: Enhance with icons for clarity

## Performance Considerations

- Each counter instance is lightweight
- Uses `useRef` to avoid unnecessary re-renders
- Spring physics is optimized for performance
- `once: true` prevents repeated animations
- Recommended: 3-8 counters per viewport
